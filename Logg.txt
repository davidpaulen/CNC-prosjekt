==================================================
OPPSETT AV HOTSPOT PÅ RASPBERRY PI
==================================================

Denne loggen forklarar kort korleis systemet er sett opp og korleis programvara fungerer.

Planen var først å bruke WiFi (Eduroam) for å kommunisere med Raspberry Pi.
Dette viste seg å vere upraktisk fordi IP-adressa endrar seg kvar gong Pi-en
koplar seg til nettet, og det blir då meir styr å finne den kvar gong.

Løysinga vart derfor å setje opp Pi-en som eit eige hotspot. Då kan ein
kople seg direkte til Pi-en utan å vere avhengig av skulenettet.
Ulempa er at ein ikkje får tilgang til internett i hotspot-modus.

For å løyse dette vart det lagt opp slik at ein kan bytte mellom hotspot
og heimenett. Då kan ein bruke internett når ein treng det, og hotspot
når ein vil kople seg direkte til Pi-en.


==================================================
PRODUKSJON AV G-KODE
==================================================

G-koden vert generert ved hjelp av OpenCV i Python. Dette blir brukt til
å lese bilete, finne konturar og lagre desse som lister med punkt.
Kvar kontur (for eksempel ein sirkel) får si eiga liste.

For å lage sjølve G-koden blir det brukt ei dobbel for-løkke der den ytre
går gjennom kvar kontur og den indre går gjennom punkta i konturen.
For kvart punkt blir det laga ei linje med G-kode. Programmet startar
med homing av aksane slik at maskina veit kvar den er.

Dette gjer det mogleg å konvertere bilete direkte til bevegelse på maskina.


------------------------------
UTFORDRINGAR OG LØYSINGAR
------------------------------

Det oppstår fleire problem når ein brukar dragknife. Kniven har eit offset
frå rotasjonspunktet, noko som gjer at den ikkje følgjer bana nøyaktig.

Dette fører til at sirklar kan bli for små, hjørne blir avrunda, og ein kan
få små avvik i starten og slutten av banar.

For å kompensere for dette har eg valt å auke diameteren på konturane litt
før G-koden blir generert. Dette gir god nok nøyaktigheit utan at det blir
for komplisert å implementere.

Eit anna problem er krappe svingar. Kniven klarer ikkje å snu retning med
ein gong, og då blir svingane runda av. For å løyse dette blir det lagt inn
ekstra bevegelse rundt punktet slik at kniven får tid til å rotere.

Den ferdige G-koden blir lagra i fila data/pakking.gcode.


==================================================
KOMMUNIKASJON: KONTROLLPANEL <-> PI5
==================================================

Kommunikasjonen skjer via seriell kommunikasjon.

På kontrollpanelet blir det sendt kommandoar med Serial.print() når ein
trykkar på ein knapp. Formatet er "CMD:<kommando>", for eksempel
"CMD:START_KUTT".

På Raspberry Pi køyrer programmet listener.py som ein systemteneste via
systemctl. Dette programmet lyttar konstant etter kommandoar.

Det er sett opp ein dictionary som mappar kommandoar til program (filstiar).
Når ein kommando blir motteken, finn programmet rett fil og køyrer den.

Programma sender også tilbake statusmeldingar som blir vist på panelet,
slik at ein kan sjå kva som skjer.


==================================================
KOMMUNIKASJON MED PRINTER
==================================================

Printeren som blir brukt er Original Prusa i3 MK3.

Planen først var å bruke OctoPrint. Det var enkelt å setje opp, men vi fekk
problem med ustabil tilkopling, truleg knytt til baudrate. Dette gjorde
løysinga lite påliteleg i praksis.

Derfor gjekk vi over til Klipper firmware. Med Klipper kan vi styre
baudrate sjølv og få meir stabil kommunikasjon mellom Pi og printer.
Vi kan også fjerne unødvendige funksjonar som ekstruder og varmeelement,
sidan vi berre brukar aksane.

Det oppstod likevel problem med flashing via Pi, fordi rebooten ikkje varte
lenge nok. Løysinga vart å flashe via laptop i staden, som fungerte bra.


==================================================
SENDE G-KODE TIL PRINTER
==================================================

I programmet start_cut.py blir G-koden først sendt frå make_gcode.py til
Klipper, der den blir lagra.

Deretter sjekkar programmet om det er kontakt med printeren og om det
allereie køyrer ein jobb. Dersom alt er i orden, blir fila pakking.gcode
starta.


==================================================
FEILHANDTERING OG NØDSTOPP
==================================================

Eit problem som oppstod var at klipper.service mista kontakt med MCU.
Då kunne ikkje Pi snakke med printeren, og jobben stoppa.

For å løyse dette måtte vi først kople oss til Pi via hotspot og restarte
klipper.service manuelt.

Seinare vart det laga eit eige program som kan restarte Klipper automatisk.
Dette fungerer også som ein nødstopp, slik at maskina kan stoppast raskt
ved behov.
